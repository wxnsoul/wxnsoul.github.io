信息收集

信息收集的意义

  • 渗透测试–安全是一个水桶效应–安全的高低取决于最薄弱的点!
    信息收集需要收集哪些信息?为啥要收集?!

whois信息

  • mark就像我申请的一个域名,在购买后实名认证,邮箱认证一堆东西,注册完了之后会有一个whois信息,这里面就是我注册域名时候留下的信息这里是属于暴露的比较少的,有时候可能会有邮箱啊,电话啊什么的

撞库攻击

  • 撞库攻击:
    密码太多了记不住,所以许多账号可能时一个密码,如果一个被破解了拿去撞其他的账号..

子域名

  • 第三条是指(DNS服务 你给他一个域名,他给你IP,所以使用字典去不断猜测子域名,看DNS服务会不会返回给你IP)
    可以去注册顶级域名,有了顶级域名就可以去无限生成其他域名
    使用阿里云和腾讯注册的需要备案,国外的则不需要

谷歌语法(内容敏感信息泄露)

  • )

web指纹



  • 就算是得知这个网站是用什么建的,cms?还是自己搭建

端口探测

  • 为什么要探测端口?

    有些危险端口开放了我们就可以尝试入侵，例如445|3306|22|1433|6379可以尝试爆破或者是使用某些端口存在漏洞的服务。而且有可能一台服务器上面不同端口代表着不同的Web网站~
    同一个端口代表什么?代表同一个服务器或者同一内网
    1433端口代表SQL server

目录扫描

  • 为什么要扫描目录?

    有些网站可能某个目录下是一个新的网站，有的时候目录扫描直接下载了压缩包源码、编辑器目录
  • 些废弃的页面(会报错)

指纹识别

  • 指纹识别(http://www.yunsee.cn/ https://s.threatbook.cn/)

    为什么要指纹识别?

    cms可能存在通杀漏洞，如果使用了CMS建站我们可以用通杀漏洞直接攻击

旁站查询

  • 为什么要查询旁站?

    旁站指的是在同一个ip上面的多个网站，如果你成功拿下旁站，运气好和主站在同一台机器上，是
    不是就是拿到了主站?如果运气不好是一个内网，我们是不是可以尝试内网渗透

C段扫描

  • C段是什么|为什么要扫描?

    例如192.168.1.1，那么192.168.1.1-192.168.1.255都属于同一个C段,有些学校或者大公司,他
    们会持有整个IP段,这个ip段中所有的ip都是那个公司的资产，拿下- -台可能有有用的信息，可能在同一内网

信息收集流程

  1. whois信息(whois网站,天眼查)
  2. 子域名(包括具有功能的IP)这个最后讲(重中之重)
  3. 端口<Nmap扫描>
  4. 旁站(同ip网站)
  5. C段120.203.75.55 以及 120 203.75.1-120.203.75.255
  6. 目录扫描(御剑)
  7. 指纹识别
  8. 内容敏感信息泄露(google hacking)
