Github使用学习

2020-05-17

杂项

#0x00前言

  • Github是一个学习的宝库啊,里面都是开源的项目,然而自己似乎只会很肤浅的一些操作..今天打算学习一波有哪些稀奇古怪的操作.用到的举例对象是一个js框架.
  • https://github.com/vuejs/vue

#0x01 Code

  • 就是存放代码的地方,主要介绍一个Branch分支
  • 这里面有着许许多多的分支节点,就是作者每进行更改一次代码就会更新一个分支.感觉理解为版本更好.
  • 这里有一个新奇的操作,就是在url后面加上/compare
  • 这个页面可以进行两个不同版本的选择,以便查看进行比较是哪些节点进行了更新
  • 可以看见两个版本中不同的地方被标注了出来,哪里进行了增删改查一目了然

#0x02 Issues

  • 这也是一个反应项目热度的地方.但不仅仅是这么简单.你可以把它理解为交流区,大家会在这里面进行提问,交流等等
  • 比如这些,会提出一些bug,一些需要改进的地方
  • 再比如这,open是现在开放还可以交流的问题,closed是管理员已经关闭的内容,可能是一些解决的问题,或者是一些被他认为是低素质的问题

#0x03 Pull requests

  • 这是一个,,,个人理解算是一个想要为这个开源项目提出改进的一个地方(做出贡献),你可以fork(复制到本地仓库)这个项目,然后自己进行修改,然后进行Pull requests,项目的核心维护人员会进行代码审查(Code Review),决定是否接受你的提议

#0x04 Projects

  • 这个工程项目就可以理解为你对这个项目接下来要做什么事情.
  • 想这就是目前有三个工程正在进行
  • 点入一个进行查看,可以看见有to do,in process,done这些列.列中有一些小任务.可以直接拖动这些小任务(to do到in process)非常的方便,就像计划任务清单一样

#0x05 Wiki

  • 这个就相当于维基百科,介绍这个项目的方方面面,有哪些工程等等
  • 可以点击create来创建

#0x06 Insights

  • 这个就是项目的统计功能
  • Contributes有哪些贡献者,Commits有哪些提交等等,总之就是各种统计信息

#0x07 Settings

  • 各种设置信息,比如说
  • 你的网站发布在哪里,域名叫啥.还有一个webhooks的功能比较强,就像是Windows上的计划任务一样,何时或者是什么样的操作后会去触发这个功能
  • trigger(触发)

#0x08 善用搜索引擎

  • in description 爬虫
  • in readme 爬虫
  • 总结
  • in:name example 名字中有“example”
  • in:readme example readme中有“example”
  • in:description example 描述中有“example”
  • stars:>1000 star>1000
  • forks:>1000 fork>1000
  • pushed:>2019-09-01 2019年9月1日后有更新的
  • language:java 用Java编写的项目

#0x0 其他
