JSP

#0x00 什么是JSP

  • JSP全称Java Server Pages，是一种动态网页开发技术。它使用JSP标签在HTML网页中插入Java代码。标签通常以<%开头以%>结束。
      • 由于服务器重启之后计数就会清零,所以可以通过将最后重启前的信息保存到文件中,就可以实现计数的功能
  • JSP是一种Java servlet，主要用于实现Java web应用程序的用户界面部分。网页开发者们通过结合HTML代码、XHTML代码、XML元素以及嵌入JSP操作和命令来编写JSP。
  • JSP通过网页表单获取用户输入数据、访问数据库及其他数据源，然后动态地创建网页。
  • JSP标签有多种功能，比如访问数据库、记录用户选择信息、访问JavaBeans组件等，还可以在不同的网页中传递控制信息和共享信息。

#0x04 输出打印

  • System.out.print()是输出在程序内部

  • 也可以使用<%=变量%>来输出这个变量
    • 类似于php中的输出变量$abc

#0x05 JSP 指令

  • JSP指令用来设置整个JSP页面相关的属性，如网页的编码方式和脚本语言。
    • Page指令
      • Page指令为容器提供当前页面的使用说明。一个JSP页面可以包含多个page指令。
      • Page指令的语法格式：
        • <%@ page attribute=”value” %>
      • 与Page指令相关的属性
    • Include指令
      • JSP可以通过include指令来包含其他文件。被包含的文件可以是JSP文件、HTML文件或文本文件。包含的文件就好像是该JSP文件的一部分，会被同时编译执行。
      • Include指令的语法格式如下：
        • <%@ include file=”relative url” %>
    • Taglib指令
      • JSP API允许用户自定义标签，一个自定义标签库就是自定义标签的集合。Taglib指令引入一个自定义标签集合的定义，包括库路径、自定义标签。
      • Taglib指令的语法：
        • <%@ taglib uri=”uri” prefix=”prefixOfTag” %>
          • uri属性确定标签库的位置，prefix属性指定标签库的前缀。

#0x06 隐藏对象

  • 输入输出对象
    • out,request,response
      • response对象有一个重定向功能可以在当前页面执行完后跳转到另一个页面,儿forward跳转不会进行forward以下的代码
  • 与属性作用域相关对象
    • pageContext,session,application
      • 当用户首次访问站点的jsp页面时,jsp容器会产生一个session对象,并分配一个唯一的标识ID,服务器就将这个作为识别客户的唯一标识.只要用户不超时访问或者关闭浏览器就会一直存!在不同的页面之间跳转也会使用同一session ID,这个就是会话维持,不用一直验证身份.
      • pageContext是一个特殊的对象,时页面中所有其他对象的集合,使用它可以获取其他对象
      • application用于保存应用程序在当前服务器上的全局数据.(服务器信息之类的)与session不同的是,所有的用户共用一个application对象
  • Servlet相关
    • page,config
      • page对象表示的时jsp页面本身,就相当于Java中的this,使用它可以调用servlet中的所有定义的类
  • 错误处理对象
    • exception
      • 如果需要使用这个对象就必须把isErrorPage对象设置为true

#0x07 EL表达式

  • 类似于js,主要用于在网页上动态的显示内容,代替Java完成复杂的脚本.
  • 语法结构
    • ${表达式}
    • 用.和[]来存取数据和对象的属性
      • 但是存取的属性名中包含一些特殊的字符串,如”.”和”?”等非字母或数字的字符就应使用[]
      • 需要动态取值也需要[]

#0x08 JSTL标签库

  • JSP标准标签库JSTL是一个实现Web程序的通用功能的定制标签库集,包括了迭代,条件判断,数据管理格式化,,XML操作以及数据库访问.
  • 通过EL和JSTL可以取代传统的向jsp页面中嵌入Java的做法,大大提高了程序的可维护性
