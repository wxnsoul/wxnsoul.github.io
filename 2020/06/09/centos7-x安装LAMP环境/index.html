centos7.x安装LAMP环境

2020-06-09

#0x00 前言#

  • 我搭建完docker之后,下载了一个centos8的镜像,然后准备复现一下emblog.但是我发现我访问我的80端口后出来的都是文件源码,百度后恍然大悟,,自己没有装LAMP环境.怪不得总是有问题…一开始自己端口转发失败就以为是自己的问题,结果就是没有咋黄LAMP环境无法访问而已
    #0x01docker踩坑#
  • 我得docker创建容器后立即就会关闭…打开也没有用.查阅资料发现Docker容器同时只能管理一个进程，如果这个进程退出那么容器也就退出了，但这不表示容器只能运行一个进程(其他进程可在后台运行)，但是要使容器不退出必须有一个前台执行的进程。
  • 解决方法：
  • 脚本中最后一个进程一定要用前台运行方式即在进程最后不加&(&表示后台运行)，否则容器会退出。
  • 直接进入容器防止容器退出

#0x02centos7.x安装LAMP环境#

  • (LAMP是指一组通常一起使用来运行动态网站或者服务器的自由软件名称首字母缩写。)
  • LAMP工作原理
  • 安装LAMP环境:
    • yum -y install httpd mariadb mariadb-server php php-mysql
    • 直接一挑明令就好了很方便
    • 解释: httpd就是apache,mariadb两个就是MySQL,被Oracle收购之后开发得新的,php-mysql就是用于存放php连接数据库得一些函数
    • 开启apache,mysql服务
      • systemctl start httpd
      • systemctl start mariadb
    • 设置开机启动
      • systemctl enable httpd
      • systemctl enable mariadb

#0x03 dockers安装LAMP环境#

  • 我在docker环境下安装了centos7.x之后安装LAMP环境.成功安装之后开启服务时报错了
  • 原因:在centos7的docker容器里面不能用service启动服务。
    • 报这个错的原因是dbus-daemon没能启动。systemctl并不是不能使用。将CMD或者entrypoint设置为/usr/sbin/init即可。docker容器会自动将dbus等服务启动起来。
  • 解决方法:
    • docker run –privileged -itd –name centos7-LAMP -p 32525:80 -v /var/www/html:/var/www/html c5d /usr/sbin/init
    • 额外添加参数–privileged,并把/bin/bash改为/usr/sbin/init
    • 并且一开始我把/usr/sbin/init放在了容器ID之前,,还报错了,必须把/usr/sbin/init放在最后
  • 然后就可以启动apache和mysql服务了,并且设置开机启动了
  • 验证
    • 终于不是php源码了
  • 创建数据库账号
    • mysqladmin -u root password “123456”
    • mysql -u root -p123456
    • 成功

#0x04将虚拟机中的docker容器映射出去#

  • 使用之前的nps
  • docker容器的80端口转发到虚拟机的32525端口,然后将32525端口转发到服务器的30001端口,访问
  • 成功了
